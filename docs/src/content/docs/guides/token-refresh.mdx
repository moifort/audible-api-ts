## Token Expiration

The `accessToken` in your credentials has a limited lifetime (typically 1 hour). The `expiresAt` field tells you when it expires.

When the token expires, API calls will fail with a `401` error.

## Refreshing the Token

```typescript
import { refreshAccessToken } from 'audible-api-ts'

const updatedCredentials = await refreshAccessToken(credentials)

// The updated credentials have a new accessToken and expiresAt
console.log('New expiration:', updatedCredentials.expiresAt)
```

The `refreshToken` is long-lived and doesn't expire in normal use.

<Aside type="tip">
  `refreshAccessToken` returns a new credentials object — it does **not** mutate the original. Remember to persist the updated credentials.
</Aside>

## Auto-Refresh Pattern

Here's a common pattern to automatically refresh before making API calls:

```typescript
import { refreshAccessToken, fetchLibrary } from 'audible-api-ts'
import type { AudibleCredentials } from 'audible-api-ts'

const ensureFreshCredentials = async (credentials: AudibleCredentials) => {
  const bufferMs = 5 * 60 * 1000 // 5 minutes buffer
  if (credentials.expiresAt.getTime() - Date.now() < bufferMs) {
    return refreshAccessToken(credentials)
  }
  return credentials
}

// Usage
let creds = loadCredentialsFromStorage()
creds = await ensureFreshCredentials(creds)

const { items } = await fetchLibrary(creds)
saveCredentialsToStorage(creds)
```

## What Gets Refreshed

| Field | Changes? |
|-------|----------|
| `accessToken` | Yes — new token |
| `expiresAt` | Yes — new expiration |
| `refreshToken` | No |
| `adpToken` | No |
| `devicePrivateKey` | No |
| `serial` | No |
| `locale` | No |
